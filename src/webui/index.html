    Nulls are coverage, not signal — sparse columns stay first-class.

    Canonical structure — unified fields

    Synonymous column names collapse onto one canonical field (type-gated, so a mislabeled column is never force-merged). Upload differently-named files and watch them line up on the same fields.

    Unified schema across all uploaded files

    This registry accumulates across every CSV you upload. Load the three customer samples (crm_contacts, sales_leads, subscribers) one after another and watch differently-named columns collapse onto the same canonical field.

    Consolidated fields — scalar vs array

    Columns sharing a canonical key are merged. Fill-pattern decides the shape: exclusive fill → scalar (survivorship), concurrent distinct values → array (a positional x_1/x_2 set becomes a bare list; a semantic x_home/x_work set keeps its label as {type, value}).
